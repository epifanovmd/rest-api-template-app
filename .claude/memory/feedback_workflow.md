Всегда разбивать задачи на этапы: анализ → подробный план → поэтапное выполнение мелкими итерациями.

**Why:** Пользователь хочет контролировать процесс, видеть план до начала работы, и получать стабильный код после каждого шага.

**How to apply:**
1. Сначала прочитать все затронутые файлы, понять scope
2. Войти в plan mode, составить пошаговый план с конкретными файлами
3. Показать план, дождаться подтверждения
4. Выполнять по одной итерации, каждая — рабочее состояние
5. После каждой итерации — lint/build проверка
6. Сообщать прогресс между итерациями
